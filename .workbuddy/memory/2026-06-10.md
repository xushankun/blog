### 关键决策与坑点

- **HTTPS 推送失败**：git-credential-manager 在非交互终端下读不到 /dev/tty。直接换 SSH (`git@github.com:xushankun/hexoBlog.git`)
- **submodule URL 双轨制**：`.gitmodules` 写 HTTPS（CI 公开仓库匿名拉取够用），本地 `.git/config` override 为 SSH（绕过本地无代理时的 HTTPS）
- **远端 master 不一致**：远端被其他自动化（AI日报）推过几个提交，因迁移已备份到 hexo-backup，对 master 用 `--force` 覆盖
- **沙盒拦截 ~/.ssh**：force push 命令需要授权 dangerouslyDisableSandbox 才能读 known_hosts
- Git Bash 下 Node.js 不认识 `/tmp`，要用 cygpath 转 Windows 路径

## CI 修复：Hugo 版本不兼容

第一次 CI 跑挂了，关键错误：

```
WARN  Module "PaperMod" is not compatible with this Hugo version: Min 0.146.0
ERROR => hugo v0.146.0 or greater is required for hugo-PaperMod to build
... partial "head.html" not found
```

原因：`HUGO_VERSION: 0.139.4` 不满足 PaperMod 最新版要求（≥ 0.146.0）。
修复：升级到 `0.147.7`，并加 `HUGO_CACHEDIR` 和 `hugo version` 打印步骤。
教训：**用主题前先看 theme.toml 的 min_version**。下次写 deploy.yml 时直接选 Hugo 最新稳定版（写本文时 0.147.x）。
